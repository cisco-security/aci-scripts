cisco-security/aci-scripts

 

The package included here allows creation of the following Tenant on APIC and ACI fabric:

 

 

BASH script wrapper for all python scripts is also included to allow step by step creation of Tenant pod1, application profile, bridge domains, EPGs, L4-7 ASA devices, their function profiles (configuration), L3outs in the ACI fabric and  those needed for ASA dynamic routing, service graphs for ASAs, and wizard based creation of the contract, filter, and applied service graphs.  Left of the WAN portion of the diagram is outside of the ACI fabric and not included in the scripting.

 

rebuild-my-pod.bash script assumes that the tenant pod1 already exists, and proceeds to delete specific items from this Tenant.  To ensure the tenant exists, you may comment out the like so:

É

   do

   #echo "^@Delete pod$i tenant items (app profile,EPGs,BDs,contract,l3outs,vrfs,L4-L7 items:  [continue]"; read go

   #python ${path}faba-tenant-delete.py $i $i

   echo "^@Create pod$i tenant app profile, EPGs, vrfs, BDs:  [continue]"; read go

É

 

You could certainly delete the tenant through APIC and run the BASH script with commented out *-tenant-delete.py section.

 

Note that included python scripts are very dependent (refer to pre-configured items) on number of settings to execute properly.  Those would have to be either matched by name or updated in the scripts to work properly.  These are items like VMM and physical domains, imported ASA device package, pre-configured vPC for ASA data plane (p10), BGP routing backplane in the fabric, etc.
